Why TurboQuantDB?

Rethinking vector databases for the era of billion-scale datasets on laptops.

⚡

Zero Training

No train() step required. Vectors are quantized and stored immediately on insert.

🗜️

10× Compression

Reduces 1536-dim float32 embeddings from 586 MB to 59 MB at 100k vectors using 2-bit quantization.

🎯

Unbiased Scoring

The combination of QR rotation and QJL guarantees unbiased inner product estimation.

🔍

Metadata Filtering

MongoDB-style filter operators ($eq, $gt, $in, $exists, $contains) on any metadata field. Bulk deletes by filter with delete_batch(where_filter=...).

Quick Start

TurboQuantDB runs in-process with a Python-native API. No daemon, no external servers.

  • Built with PyO3 and Maturin — no daemon required
  • Write-ahead log (WAL) + CRC32 crash recovery
  • LangChain, ChromaDB, and LanceDB compatible
  • Parameterless reopen from stored manifest
example.py
import numpy as np
from tqdb import Database

# Create database (dimension required on first open)
db = Database.open("./my_db", dimension=1536, bits=4, rerank=True)

# Insert documents and metadata
db.insert("doc-1",
    np.random.randn(1536).astype("f4"),
    metadata={"topic": "ml", "year": 2026},
    document="Machine learning intro"
)
db.close()

# Reopen later — no need to specify dimension or other params
db = Database.open("./my_db")

# Search with MongoDB-style filter
results = db.search(
    np.random.randn(1536).astype("f4"),
    top_k=5,
    filter={"year": {"$gte": 2026}}
)
for r in results:
    print(r["id"], r["score"], r["document"])

# Bulk delete by metadata filter
deleted = db.delete_batch(where_filter={"year": {"$lt": 2020}})

The "Turbo" Trick: Random Rotation

Real-world data can be highly skewed. TurboQuant applies a random rotation matrix to transform coordinates into a predictable Normal distribution.
